Overview

Optimized Test Flow Automation (OTFA) Tool is a custom framework, which uses the Selenium API and the feature file generated from CTD to run tests in a specified browser. OTFA is powered by BDD style of defining test cases from Cucumber/Gherkin and web automation from Selenium. It leverages a powerful learning and pattern based Discovery and Mapping Engine to identify the steps and application objects based on the application type, scenario, and the current context.

What's new in this release

OTFA V3.0 has the following new features:

  • Supports execution of test cases on mobile applications:
    • Chrome plugin to capture objects.
    • Supports execution of feature file for mobile enabled web application on any android device.
  •  Grid management:
    • Ability to add or delete test machines on which the test cases can be run.
  • Other enhancements:
    • Ability to delete cookies before execution in both web and mobile versions.
    • Support for background section of the feature file.
    • Enhanced object capture to automatically map objects for the same application repeated across different feature files, resulting in reduced execution time.
    • Additional validation step for object mapping to verify page content before execution.
    • Overcome limitations with XPath with the ability to capture an object by its CSS, ID, name, or XPath.
    • Enhanced user interface for Test Execution and Feature File Import tabs.

     Prerequisites

  • Java 1.7
  • Google Chrome and Firefox browsers
  • Open Page with Chrome add-on for the Firefox browser to invoke Chrome browser from the Firefox browser.
  • Following plugins for Firefox browser, in order to install the Object Builder:
    • Firefox version 1.5, or higher, up to version 48.
    • Firebug version 2.0.14.
